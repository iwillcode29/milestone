## 4. Data — IndexedDB ล้วน ไม่มี SQL

```ts
type Result = {
  bib: string;          // '001'–'025' zero-padded
  name: string;
  p1_sec: number;       // เพซช่วง 1 Road (วินาที/กม.)
  p2_sec: number;       // เพซช่วง 2 Trail
  p3_sec: number;       // เพซช่วง 3 Hilly
  act_sec: number;      // Activity Time
  d1_km?: number;       // ระยะจาก GPS — ไม่คิดคะแนน ใช้ติดธงเตือน
  d2_km?: number;
  d3_km?: number;
  recorded_at: number;  // Date.now() — ใช้ตัดสินเสมอชั้นสุดท้าย
};

// เก็บ 3 key พอ
// 'teams'   → Team[]   (seed จาก CSV ตอนตั้งค่า)
// 'results' → Record<bib, Result>
// 'config'  → Config
```

**Config (แก้ได้ในหน้า settings):**

| key | ค่า | ที่มา |
|---|---|---|
| `target_p1_sec` | `503` | 8:23 — เดือนที่จัด/ครั้งที่จัด |
| `target_p2_sec` | `675` | 11:15 — ปีครบรอบ/ปีก่อตั้ง Amazfit |
| `target_p3_sec` | `645` | 10:45 — เวลาปล่อยตัว Chamonix 17.45 |
| `target_act_sec` | `2805` | 46:45 — cut off Chamonix 46hrs45mins |
| `weight_pace` | `1.0` | |
| `weight_act` | `0.2` | ตั้งต่ำเพราะหน่วยเวลารวมใหญ่กว่าเพซ ~10 เท่า |
| `gps_tolerance_pct` | `10` | |

**⚠️ เรียก `navigator.storage.persist()` ตอนเปิดแอปครั้งแรก** — บอกเบราว์เซอร์ว่าอย่าล้างข้อมูลนี้ทิ้ง ถ้าไม่เรียก iOS Safari มีสิทธิ์เคลียร์ทิ้งได้

## 5. `src/lib/scoring.ts` — โมดูลเดียว มี unit test

```ts
export const dev = (actual: number, target: number) => Math.abs(actual - target);

export function scoreOf(r: Result, c: Config) {
  const d1 = dev(r.p1_sec, c.target_p1_sec);
  const d2 = dev(r.p2_sec, c.target_p2_sec);
  const d3 = dev(r.p3_sec, c.target_p3_sec);
  const da = dev(r.act_sec, c.target_act_sec);
  return {
    d1, d2, d3, da,
    maxDev: Math.max(d1, d2, d3),
    total: (d1 + d2 + d3) * c.weight_pace + da * c.weight_act,
  };
}

// คะแนนน้อยชนะ
// ตัดสินเสมอ: 1) ช่วงที่พลาดมากสุด พลาดน้อยกว่าชนะ
//             2) ใกล้เป้า Trail มากกว่าชนะ
//             3) เข้าเส้นก่อนชนะ
export const rank = (rs: Scored[]) => [...rs].sort((a, b) =>
  a.total - b.total || a.maxDev - b.maxDev || a.d2 - b.d2 || a.recorded_at - b.recorded_at
);
```

**ทำไมต้องมี tiebreak:** ทดสอบกับข้อมูลจำลอง 20 ทีมแล้ว ถ้าคิดจากเพซล้วนไม่มี Activity Time จะเสมอทันที (2 ทีมได้ 68 เท่ากัน) พอใส่ Activity Time × 0.2 คะแนนมีทศนิยม ไม่เสมอเลย — แต่ยังต้องมี tiebreak กันเหนียว

## 7. หน้ากรอกผล — สเปคละเอียด

### Signature: ทำหน้าจอให้เป็นกระจกสะท้อนนาฬิกา

หน้า **Lap Details** ของ Amazfit วาง 3 แถว คอลัมน์ `เวลา · ระยะ · เพซ` → แอปต้องวาง **3 แถว เรียงคอลัมน์เดียวกันเป๊ะ** staff จะได้ไม่ต้องแปลงในหัวว่าแถวไหนของนาฬิกาตรงกับช่องไหนของแอป

```
┌─────────────────────────────────┐
│  001  ขาแรงกาแล                 │
├─────────────────────────────────┤
│       ระยะ (กม.)      เพซ       │
│  ①     [ 1.52 ]     [  8:23 ]  │  ← Road   เป้า 8:23
│  ②     [ 1.19 ]     [ 11:15 ]  │  ← Trail  เป้า 11:15
│  ③     [ 1.31 ]     [ 10:45 ]  │  ← Hilly  เป้า 10:45
├─────────────────────────────────┤
│  Activity Time      [ 46:45 ]  │
├─────────────────────────────────┤
│         ┌───┬───┬───┐           │
│         │ 1 │ 2 │ 3 │           │
│         ├───┼───┼───┤           │  numpad ของเราเอง
│         │ 4 │ 5 │ 6 │           │  ห้ามใช้คีย์บอร์ดเครื่อง
│         ├───┼───┼───┤           │
│         │ 7 │ 8 │ 9 │           │
│         ├───┼───┼───┤           │
│         │ ⌫ │ 0 │ → │           │
│         └───┴───┴───┘           │
└─────────────────────────────────┘
```

### กติกาการกรอก

- **พิมพ์เลขล้วน เติมจากขวา แบบนาฬิกาจับเวลา** — กด `8` `2` `3` → โชว์ `8:23` ทันที ไม่ต้องกด `:`
- **numpad ของเราเอง** ห้ามใช้ native keyboard — มันช้า เด้ง viewport มีปุ่มรกที่ไม่ได้ใช้
- ปุ่มขั้นต่ำ **64×64px** ระยะห่าง ≥ 8px
- แตะช่องไหน = แก้ช่องนั้น · `→` เลื่อนช่องถัดไป
- ช่องระยะทศนิยม 2 ตำแหน่ง (`152` → `1.52`) · **ข้ามได้** ถ้าคนเข้าเส้นรัวๆ กลับมาเติมทีหลัง
- ค้นหา bib พิมพ์ `1` ต้องเจอ `001`

### หน้ายืนยัน (ห้ามข้าม)

ตัวเลข ≥ 32px + ส่วนต่างมีสี + คะแนนที่จะได้ + ปุ่ม `แก้ไข` / `บันทึก`

```
ช่วง 1    8:31   ห่าง  +8 วิ   🟢
ช่วง 2   11:38   ห่าง +23 วิ   🟡
ช่วง 3   10:43   ห่าง  −2 วิ   🟢
Activity 47:27   ห่าง +42 วิ   🟡
──────────────────────────────
คะแนนรวม  41.4
```

## 8. ตรวจสอบความสมเหตุสมผล

**บล็อก (hard):** เพซนอกช่วง `4:00`–`25:00` · Activity Time นอกช่วง `25:00`–`90:00`

**เตือนแต่ผ่านได้ (modal ยืนยันซ้ำ):**
- ห่างเป้าเกิน 90 วิ/กม. → *"ห่างเป้ามาก ตรวจสอบอีกครั้ง"* (มักเป็นการพิมพ์สลับหลัก)
- Activity Time ไม่สอดคล้องกับเพซ 3 ช่วง → น่าจะจดผิดจอ

**ธง GPS (ไม่บล็อก):** เทียบ `d1/d2/d3` กับ median ของทุกทีมที่กรอกแล้ว เกิน 10% → ติดธง `⚠️` ในหน้ารายชื่อ
เนื่องจากทุกทีมใช้นาฬิการุ่นเดียวกัน ธงนี้แทบไม่ขึ้น — แต่ถ้าขึ้นแปลว่าเครื่องนั้นมีปัญหาจริง ต้องรู้ก่อนประกาศผล

## 9. ⚠️ ความเสี่ยงอันดับ 1 — เครื่องเดียว ไม่มี server

**ถ้ามือถือหาย/พัง/แบตหมด/เผลอกด "ล้างข้อมูลเว็บไซต์" → ผลแข่งหายหมด กู้ไม่ได้**

ไม่มี server = ไม่มีตาข่ายรองรับ ต้องแก้ด้วยวินัยแทน:

1. **`navigator.storage.persist()`** ตอนเปิดแอปครั้งแรก — กันเบราว์เซอร์เคลียร์ทิ้งเอง
2. **Auto-backup ทุก 5 รายการ** — เซฟไฟล์ `.json` ลงเครื่องอัตโนมัติ ไม่ต้องถาม
3. **ปุ่ม `Export CSV` ค้างบน header ทุกหน้า** — ไม่ซ่อนในเมนู · กดแล้วได้ไฟล์ที่ยัดเข้า Excel เดิมได้ทันที
4. **แถบเตือนสีส้ม** ถ้ายังไม่ export เกิน 10 รายการ: *"ยังไม่ได้สำรอง 10 รายการ"*
5. **หน้างาน: ต่อ powerbank ไว้ตลอด** ปิด auto-lock ไม่ต้องประหยัดแบต

> **ถ้ารับความเสี่ยงนี้ไม่ได้** ทางเลือกถูกที่สุดคือเพิ่ม Vercel serverless function 1 ตัว + **Vercel Blob** ยิง snapshot JSON ทั้งก้อนขึ้นไปหลังบันทึกทุกครั้ง (~20 บรรทัด ไม่มี schema ไม่มี migration ไม่มี sync logic — แค่ overwrite ทับไปเรื่อยๆ ล้มเหลวก็ช่างมัน เพราะ IndexedDB คือตัวจริงอยู่แล้ว)
>
> ผมแนะนำให้ทำครับ — ราคา 20 บรรทัด แลกกับการไม่ต้องนั่งเสียวทั้งงาน

## 13. ไม่ทำ (ตัดสินใจแล้ว)

- ❌ PIN / ระบบ user — เครื่องอยู่ในมือตัวเอง

- ❌ Dark mode
- ❌ ถ่ายรูปแนบ — ถ้าอยากได้หลักฐาน ให้ staff ถ่ายด้วยแอปกล้องปกติแยกไว้ ง่ายกว่าและไม่กินพื้นที่แอป
